git报错与解决方案

发表于 2023-11-17 更新于 2023-11-23 分类于 git 阅读次数：

报错1

当pull遇到如下错误时

github上的肯定是最新的代码，可能的原因是本机的代码添加了无关紧要的注释，且添加前没有pull，造成的冲突导致的。

此时的解决方案是将remote的main分支，pull到本机新建的main branch中，然后切换到main分支，将原分支master删除。此做法只适合本机没有更新代码的情况。
