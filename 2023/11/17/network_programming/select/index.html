epoll

epoll从Linux的2.5.44版本内核引入的。

epoll例程即创建出来的文件描述符保存空间

实现epoll必要的结构体和函数

epoll_event结构体用于保存发生事件文件描述符集合

epoll_event的成员events中可以用与

  • EPOLLIN ：表示对应的文件描述符可以读（包括对端SOCKET正常关闭）；

  • EPOLLOUT：表示对应的文件描述符可以写；

  • EPOLLPRI：表示对应的文件描述符有紧急的数据可读（这里应该表示有带外数据到来）；

  • EPOLLERR：表示对应的文件描述符发生错误；

  • EPOLLHUP：表示对应的文件描述符被挂断；

  • EPOLLET： 将EPOLL设为边缘触发(Edge Triggered)模式，这是相对于水平触发(Level Triggered)来说的。

  • EPOLLONESHOT：只监听一次事件，当监听完这次事件之后，如果还需要继续监听这个socket的话，需要再次把这个socket加入到EPOLL队列里

1、epoll_create：创建保存epoll文件描述符空间；

  • 参数：指定epoll例程的大小（Linux2.6.8后OS将自动调整大小，可以忽略size参数）。
  • 返回值：成功时返回epoll文件描述符，失败时返回-1。

2、epoll_ctl：向空间注册并注销文件描述符；

用于在epoll例程注册 监视对象文件描述符

  • 参数
    • epfd：epoll例程的文件描述符
    • op：对监控对象的的注册、删除、更改等操作。
      • EPOLL_CTRL_ADD
      • EPOLL_CTRL_DEL：传递时，同时要向第四个参数event传递NULL，
      • EPOLL_CTRL_MOD
    • fd：需要注册的监视对象描述符
    • event：监视对象的事件类型
  • 返回值：成功0，失败-1

3、epoll_wait：与select相同，等待文件描述符发生变化。

  • 参数
    • epfd：epoll实例文件描述符
    • event：epoll_event结构体地址
    • maxevents：第二个参数中可以保存的最大事件数
    • timeout：以毫秒为单位的等待时间，-1时一直等待。
  • 返回值：成功返回事件的文件描述符数，失败返回-1。

条件触发和边缘触发

条件触发（Level Trigger）：只要输入缓冲有数据，就会一直通知该事件。

边缘触发（Edge Trigger）：输入缓冲仅在收到数据时注册一次该事件。
